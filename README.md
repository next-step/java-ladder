# 사다리 게임
## 진행 방법
* 사다리 게임 게임 요구사항을 파악한다.
* 요구사항에 대한 구현을 완료한 후 자신의 github 아이디에 해당하는 브랜치에 Pull Request(이하 PR)를 통해 코드 리뷰 요청을 한다.
* 코드 리뷰 피드백에 대한 개선 작업을 하고 다시 PUSH한다.
* 모든 피드백을 완료하면 다음 단계를 도전하고 앞의 과정을 반복한다.

## 온라인 코드 리뷰 과정
* [텍스트와 이미지로 살펴보는 온라인 코드 리뷰 과정](https://github.com/nextstep-step/nextstep-docs/tree/master/codereview)

## 🚀 2단계 - 사다리(생성)
### ✅ Model (도메인 로직 및 비즈니스 규칙)
- [x] `Player`는 이름을 갖고, 이름은 5자 이하, 비어있지 않고, 숫자만/특수문자 포함 불가하도록 제한한다.
- [x] `Players`는 사용자 입력 문자열을 파싱하여 `Player` 리스트로 구성한다.
- [x] `Point`는 해당 위치에서 오른쪽으로 가로줄이 존재하는지를 표현한다.
- [x] `Line`은 플레이어 수 만큼의 `Point` 리스트를 생성하고, 연속된 가로줄을 허용하지 않도록 한다.
- [x] `Ladder`는 높이만큼의 `Line` 리스트를 갖고, 유효하지 않은 높이나 너비(플레이어 수)에 대한 검증을 수행한다.

### ✅ View (입출력 담당)
- [x] 사용자로부터 참여자 이름을 쉼표(,)로 구분하여 입력받는다.
- [x] 사용자로부터 사다리의 최대 높이를 입력받는다.
- [x] 사다리 상단에 참여자 이름을 5자 폭으로 출력한다.
- [x] 사다리를 출력할 때, 오른쪽으로 가로줄이 연결된 경우만 `-----`로 표현한다.

### ✅ Controller (입력 → 도메인 처리 → 출력 흐름 제어)
- [x] 사용자 입력을 받아 `Players`, `Ladder`를 생성하고 도메인 객체에 위임한다.
- [x] 생성된 `Players`, `Ladder` 객체를 View로 전달하여 출력한다.
