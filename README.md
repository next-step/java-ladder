# 사다리 게임
## 진행 방법
* 사다리 게임 게임 요구사항을 파악한다.
* 요구사항에 대한 구현을 완료한 후 자신의 github 아이디에 해당하는 브랜치에 Pull Request(이하 PR)를 통해 코드 리뷰 요청을 한다.
* 코드 리뷰 피드백에 대한 개선 작업을 하고 다시 PUSH한다.
* 모든 피드백을 완료하면 다음 단계를 도전하고 앞의 과정을 반복한다.

## 온라인 코드 리뷰 과정
* [텍스트와 이미지로 살펴보는 온라인 코드 리뷰 과정](https://github.com/nextstep-step/nextstep-docs/tree/master/codereview)

## 기능 구현
- 익명 클래스 람다로 전환
- 람다를 활용해 중복 제거
- 스트림(Stream) map, filter, reduce 구현
- Optional 
  - 조건에 따른 반환
  - 값을 반환 
  - exception 처리

# 사다리 기능 구현

## 기능 명세서 

## Persons 클래스 - 이름들 구분
1. [✅] 참여할 사람 이름 구분자로 나누는 로직
2. [✅] 참여자가 최소 인원(2명) 이상인지 확인하여, 조건을 만족하지 않으면 예외를 발생
3. [✅] 참여자 정보 제공
4. [✅] 특정 이름을 가진 Person 객체를 검색

## Person 클래스 - 이름
1. [✅] 참여할 사람 5글자 validate 체크 로직
2. [✅] 이름이 "all"과 같거나, 최대 글자 수(5글자)를 초과하지 않도록 검사
3. [✅] 현재 위치를 반환하고, 게임 결과에 따라 위치를 업데이트

## Line 클래스 
1. [✅] Points 클래스는 Boolean 값을 받아 각 위치의 점을 Point 객체로 초기화
2. [✅]  연속된 true 값이 존재하지 않도록 유효성을 검사
3. [✅] 주어진 height만큼의 라인을 생성하며, weight와 strategy에 따라 각 라인을 초기화
4. [✅] 사다리의 모든 라인(lines) 목록을 반환
5. [✅] lines를 순회하면서 Line의 move 메서드를 호출하여 위치를 업데이트
6. [✅] 특정 위치에서 좌측이나 우측으로 이동할 수 있는지 여부를 확인

## Ladder 클래스
1. [✅] 사다리 개수 validate 체크 로직 
2. [✅] 처음 부터 라인 판단 리스트 List 로직 
3. [✅] 라인정보 반환 로직 

## LadderNumber 클래스 
1. [✅] 숫자 validate 검증 로직
2. [✅] 숫자 반환 로직


## LadderPlayResult 클래스
1. [✅] 사다리 게임의 결과를 저장하고 조회

## LadderResult 클래스 
1. [✅] 결과 데이터를 Map 형태로 저장하여 각 인덱스에 해당하는 실행 결과 관리
2. [✅] 주어진 결과의 개수가 참여자 수와 일치하는지 확인
3. [✅] 인덱스를 통해 특정 위치의 실행 결과 가져옴

## Point 클래스
1. [✅] 현재 Point 객체가 연결 상태(true 또는 false)인지 확인

## Points 클래스
1. [✅] addHorizontal 메서드를 통해 전략에 따라 새로운 Point 객체를 추가
2. [✅] 이동 가능 여부
3. [✅] 지점 목록 제공

## Position 클래스
1. [✅] 위치 값이 0보다 작지 않은지 확인
2. [✅] 현재 위치 값을 외부에서 조회할 수 있도록 반환

## PositiveNumber 클래스
1. [✅] 주어진 숫자가 최소값(2)을 만족하는지 검사하고, 유효하지 않으면 예외를 발생

## InputView 클래스
1. [✅] 참가자 이름 입력
2. [✅] 최대 사다리 높이 입력

## ResultView 클래스
1. [✅] 사람이름 결과 반환
2. [✅] 사다리 이름 결과 반환
