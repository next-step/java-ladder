## 기능 요구사항
- 사다리 게임에 참여하는 사람에 이름을 최대5글자까지 부여할 수 있다. 사다리를 출력할 때 사람 이름도 같이 출력한다.
- 사람 이름은 쉼표(,)를 기준으로 구분한다.
- 사람 이름을 5자 기준으로 출력하기 때문에 사다리 폭도 넓어져야 한다.
- 사다리 타기가 정상적으로 동작하려면 라인이 겹치지 않도록 해야 한다.
- |-----|-----| 모양과 같이 가로 라인이 겹치는 경우 어느 방향으로 이동할지 결정할 수 없다.
### 실행 결과
- 위 요구사항에 따라 4명의 사람을 위한 5개 높이 사다리를 만들 경우, 프로그램을 실행한 결과는 다음과 같다.
```
참여할 사람 이름을 입력하세요. (이름은 쉼표(,)로 구분하세요)
pobi,honux,crong,jk

최대 사다리 높이는 몇 개인가요?
5

실행결과

pobi  honux crong   jk 
    |-----|     |-----|
    |     |-----|     |
    |-----|     |     |
    |     |-----|     |
    |-----|     |-----|
```

## 메모
- 사람 이름 최대 5글자
- 입력 받을 때 사람 이름은 쉼표로 구분
- 사다리를 어떻게 관리하고 생성하지
  - 사다리 (Ladder)
    - 각 행 (Row)
      - 발판 (Bridge)
- 사다리 생성 로직
  1. 각 행 별로 진행 (입력받은 사다리 높이 만큼 반복)
  2. Bridge 를 몇 개 설치할지 Random 하게 추출 → 단, 각 행 별로 설치 가능한 Bridge 수의 범위는 0~(player/2)
  3. (2)에서 뽑은 설치하고자 하는 Bridge 의 수 만큼 시작점 `(Bridge의 왼쪽)` 을 Random 하게 뽑는다. → 시작점의 범위는 0~player-1
     1. 시작점 조합이 유효한지 검사하며, 유효하지 않다면 다시 3번의 과정을 진행한다. → 겹치면 정상적인 사다리게임 진행이 안되기 때문이다.
     2. 시작점 조합이 유효하다면 Row 에 할당해준다.
- Row 에 너무 많은 메서드가 있는 듯 (특히, Bridge 생성관련 부분에 대해서. 이 부분은 다른 클래스로 역할을 위임해도 좋을듯)
